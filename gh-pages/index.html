Key Features

📊

Project Scoring (0-100)

Weighted formula: sell-through (40%) + PSF momentum (30%) + developer (20%) + handover (10%). Fully explainable, no ML.

📈

IRR Calculator

Compare payment plans side-by-side. See how down payment structure affects annualised returns. Sensitivity table with 8 scenarios.

🔔

Real-time Alerts

Score drops, PSF spikes, handover delays. Hourly checks via Edge Functions. Email + in-app notifications.

🏢

Developer Scorecards

RERA complaints, on-time delivery %, quality ratings, historical ROI. Know who builds on time.

Database Schema

11 tables, all with RLS. Public read for market data, user-scoped for personal data.

erDiagram
    developers ||--o{ projects : "builds"
    projects ||--o{ psf_history : "tracks"
    projects ||--o{ payment_plans : "offers"
    projects ||--o{ dld_transactions : "matched"
    projects ||--o{ score_snapshots : "scored"
    user_profiles ||--o{ watchlist : "watches"
    user_profiles ||--o{ alerts_log : "receives"
    user_profiles ||--|{ alert_preferences : "configures"
    projects ||--o{ watchlist : "watched"

    developers { uuid id PK string name int developer_score }
    projects { uuid id PK string name string area int score int current_psf }
    psf_history { uuid id PK date recorded_date int psf string source }
    payment_plans { uuid id PK string name float down_payment_pct }
    dld_transactions { uuid id PK date transaction_date int transaction_value }
    user_profiles { uuid id PK string email enum subscription_tier }
    watchlist { uuid id PK uuid user_id uuid project_id }
    alerts_log { uuid id PK enum alert_type string title bool is_read }
    alert_preferences { uuid user_id PK bool email_alerts bool weekly_digest }
    score_snapshots { uuid id PK date score_date int score }

IRR Calculator

Simplified annualised ROI on cash invested during construction. The key paid feature.
flowchart LR
    A["Unit Price"] --> C["IRR Engine"]
    B["Exit PSF"] --> C
    D["Hold Years"] --> C
    E["Payment Plan"] --> C
    C --> F["Estimated IRR %"]
    C --> G["Net Gain AED"]
    C --> H["Sensitivity x8"]

Key Insight

Lower down payment = less cash at risk = higher IRR. Comparing payment plans is the killer feature because the same property can have wildly different returns depending on the payment structure.

Sensitivity Table

8 exit PSF scenarios: -10%, -5%, 0%, +5%, +10%, +15%, +20%, +30% of current PSF. Shows IRR and gain at each level.
